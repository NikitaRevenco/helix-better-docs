description: A list of all of the available keymappings in Helix

:::note

Mappings marked (**LSP**) require an active language server for the file.

:::

:::note

Mappings marked (**TS**) require a tree-sitter grammar for the file type.

:::

:::caution

Some terminals' default key mappings conflict with Helix's. If any of the mappings described on this page do not work as expected, check your terminal's mappings to ensure they do not conflict. See the [wiki](https://github.com/helix-editor/helix/wiki/Terminal-Support) for known conflicts.

:::

## Normal mode

Normal mode is the default mode when you launch helix. You can return to it from other modes by pressing the <kbd>Escape</kbd> key.

### Search

Search commands all operate on the <kbd>/</kbd> register by default. To use a different register, use <kbd>"</kbd> + `<char>`.

| Key           | Description                                 | Command            |
| ------------- | ------------------------------------------- | ------------------ |
| <kbd>/</kbd>  | Search for regex pattern                    | `search`           |
| <kbd>?</kbd>  | Search for previous pattern                 | `rsearch`          |
| <kbd>n</kbd>  | Select next search match                    | `search_next`      |
| <kbd>N</kbd>  | Select previous search match                | `search_prev`      |
| <kbd>\*</kbd> | Use current selection as the search pattern | `search_selection` |

#### View mode

Accessed by typing <kbd>z</kbd> in [normal mode](#normal-mode).

View mode is intended for scrolling and manipulating the view without changing
the selection. The "sticky" variant of this mode (accessed by typing <kbd>Z</kbd> in
normal mode) is persistent and can be exited using the escape key. This is
useful when you're simply looking over text and not actively editing it.

| Key                                                   | Description                                               | Command                 |
| ----------------------------------------------------- | --------------------------------------------------------- | ----------------------- |
| <kbd>z</kbd> or <kbd>c</kbd>                          | Vertically center the line                                | `align_view_center`     |
| <kbd>t</kbd>                                          | Align the line to the top of the screen                   | `align_view_top`        |
| <kbd>b</kbd>                                          | Align the line to the bottom of the screen                | `align_view_bottom`     |
| <kbd>m</kbd>                                          | Align the line to the middle of the screen (horizontally) | `align_view_middle`     |
| <kbd>j</kbd> or <kbd>down</kbd>                       | Scroll the view downwards                                 | `scroll_down`           |
| <kbd>k</kbd> or <kbd>up</kbd>                         | Scroll the view upwards                                   | `scroll_up`             |
| <kbd>Ctrl</kbd> + <kbd>f</kbd> or <kbd>PageDown</kbd> | Move page down                                            | `page_down`             |
| <kbd>Ctrl</kbd> + <kbd>b</kbd> or <kbd>PageUp</kbd>   | Move page up                                              | `page_up`               |
| <kbd>Ctrl</kbd> + <kbd>u</kbd>                        | Move cursor and page half page up                         | `page_cursor_half_up`   |
| <kbd>Ctrl</kbd> + <kbd>d</kbd>                        | Move cursor and page half page down                       | `page_cursor_half_down` |

## Insert mode

Accessed by typing <kbd>i</kbd> in [normal mode](#normal-mode).

Insert mode bindings are minimal by default. Helix is designed to
be a modal editor, and this is reflected in the user experience and internal
mechanics. Changes to the text are only saved for undos when
escaping from insert mode to normal mode.

:::tip

New users are strongly encouraged to learn the modal editing paradigm to get the smoothest experience.

:::

| Key                                                                                           | Description               | Command                  |
| --------------------------------------------------------------------------------------------- | ------------------------- | ------------------------ |
| <kbd>Escape</kbd>                                                                             | Switch to normal mode     | `normal_mode`            |

## Select / extend mode

Accessed by typing <kbd>v</kbd> in [normal mode](#normal-mode).

Select mode echoes Normal mode, but changes any movements to extend
selections rather than replace them. Goto motions are also changed to
extend, so that <kbd>vgl</kbd>, for example, extends the selection to the end of
the line.

Search is also affected. By default, <kbd>n</kbd> and <kbd>N</kbd> will remove the current
selection and select the next instance of the search term. Toggling this
mode before pressing <kbd>n</kbd> or <kbd>N</kbd> makes it possible to keep the current
selection. Toggling it on and off during your iterative searching allows
you to selectively add search terms to your selections.
